# git_cmd.txt
git이란 주로 텍스트 파일의 버전을 관리하기 위한 버전관리시스템이다.
git으로 관리하는 하나의 프로젝트 단위를 리포지토리(repository)라고 부르며, 내 컴퓨터에 존재하는 폴더를 하나의 리포지토리로 만들 수 있다.
이렇게 만든 리포지토리는 로컬 리포라고 부른다.

# 현재 폴더에 대해서 리포지토리 정보를 확인하는 명령어
git status
-> .git 숨김폴더가 없다면 해당 폴더는 리포가 아닌 것이다.
# 이러한 리포를 처음 새로 생성하는 명령어
git init
-> .git 숨김폴더를 생성하고 하나의 리포지토리로 삼는다.

# 생성, 수정, 삭제한 기록을 확정짓는 행위: commit
# 이러한 커밋을 하기 위해 파일을 기록에 추가하는 것
# -> staging
# 파일을 스테이징하는 명령어
git add <파일명>

# 하나의 버전을 확정짓는 명령어
git commit -m "해당 버전에 대한 정보 메시지"
-> 커밋한 내용에 대해서 메시지, 코멘트를 남길 수 있다.

# 모든 변경된 파일을 스테이징하는 명령어
git add .

===================
# github에 처음으로 리포지토리를 올린 뒤, 이후의 수정사항을
# 깃허브에 올리려면 이전의 커밋까지 동일하게 진행한 뒤
# 변경사항을 원격 리포지토리에 업로드, 싱크하는 명령어
git push origin main
# 이때 origin의 main 브랜치 하나만 remote에 연결되어 있다면
# 간단하게 git push만 입력해도 동작한다.

# 다른 로컬 리포에서 원격 리포의 변경사항을 업데이트 받으려면 땡겨와야 한다.
# 원격 리포 -> 로컬 리포 최신 변경사항 적용하는 명령어
git pull

# 원격 github와 같은 사이트에 올라간 리포를 로컬에 복제해 받아오는 것: clone
# 복제 명령어
git clone <원격저장소url> .
-> 마지막의 마침표.는 현재 git bash가 열린 폴더에 해당 리포 파일을 직접 받는 것을 의미한다. 마침표.를 생략하면 github에 작성된 리포 이름으로 폴더가 생성된 뒤 내용이 복제된다.
